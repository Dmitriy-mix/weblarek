# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.ф

## Модели данных

Модели данных отвечают за хранение и работу с данными в приложении. Ниже описаны три класса, реализующие эту функциональность.

### 1. Класс `ProductCatalog` (каталог товаров)

**Назначение и зона ответственности**  
Отвечает за хранение списка всех доступных товаров, а также за управление выбранным для детального просмотра товаром.

**Конструктор**  
Не принимает параметров. При создании экземпляра инициализирует пустой массив товаров и `null` для выбранного товара.

**Поля класса**

| Поле | Тип | Описание |
|------|-----|----------|
| `_products` | `IProduct[]` | Массив всех товаров, доступных в приложении |
| `_selectedProduct` | `IProduct \| null` | Товар, выбранный для подробного отображения |

**Методы класса**

| Метод | Параметры | Возвращаемое значение | Описание |
|-------|-----------|----------------------|----------|
| `setProducts(products)` | `products: IProduct[]` | `void` | Сохраняет переданный массив товаров в поле `_products` |
| `getProducts()` | нет | `IProduct[]` | Возвращает массив всех товаров |
| `getProductById(id)` | `id: string` | `IProduct \| undefined` | Возвращает товар по указанному идентификатору |
| `setSelectedProduct(product)` | `product: IProduct` | `void` | Сохраняет товар для подробного отображения |
| `getSelectedProduct()` | нет | `IProduct \| null` | Возвращает выбранный для просмотра товар |

**Интерфейс**

```typescript
interface IProduct {
  id: string;          // уникальный идентификатор товара
  description: string; // подробное описание
  image: string;       // URL изображения
  title: string;       // название товара
  category: string;    // категория (например, "электроника")
  price: number | null;// цена (null означает бесплатно или не указана)
}
```

### 2. Класс `Cart` (корзина)

**Назначение и зона ответственности**  
Управляет списком товаров, которые пользователь добавил для покупки. Обеспечивает добавление, удаление, подсчёт стоимости и количества товаров в корзине.

**Конструктор**  
Не принимает параметров. Инициализирует пустой массив товаров.

**Поля класса**

| Поле | Тип | Описание |
|------|-----|----------|
| `_items` | `IProduct[]` | Массив товаров, добавленных в корзину |

**Методы класса**

| Метод | Параметры | Возвращаемое значение | Описание |
|-------|-----------|----------------------|----------|
| `getItems()` | нет | `IProduct[]` | Возвращает массив товаров в корзине |
| `addItem(product)` | `product: IProduct` | `void` | Добавляет товар в корзину |
| `removeItem(productId)` | `productId: string` | `void` | Удаляет товар с указанным `id` из корзины |
| `clear()` | нет | `void` | Очищает корзину (удаляет все товары) |
| `getTotalPrice()` | нет | `number` | Возвращает сумму цен всех товаров в корзине (цены `null` трактуются как 0) |
| `getItemCount()` | нет | `number` | Возвращает количество товаров в корзине |
| `contains(productId)` | `productId: string` | `boolean` | Проверяет, присутствует ли товар с указанным `id` в корзине |

```typescript
interface ICart {
  getItems(): IProduct[];               // Получить все товары в корзине
  addItem(product: IProduct): void;     // Добавить товар
  removeItem(productId: string): void;  // Удалить товар по id
  clear(): void;                        // Очистить корзину
  getTotalPrice(): number;              // Общая стоимость товаров в корзине
  getItemCount(): number;               // Количество товаров
  contains(productId: string): boolean; // Проверить наличие товара по id
}
```

### 3. Класс `Buyer` (покупатель)

**Назначение и зона ответственности**  
Хранит и предоставляет доступ к данным покупателя, необходимым для оформления заказа. Выполняет валидацию полей в соответствии с функциональными требованиями.

**Конструктор**  
Не принимает параметров. Инициализирует поля значениями по умолчанию (пустые строки для текстовых полей, для `payment` - значение по умолчанию `'cash'`).

**Поля класса**

| Поле | Тип | Описание |
|------|-----|----------|
| `_payment` | `TPayment` | Выбранный способ оплаты |
| `_email` | `string` | Адрес электронной почты |
| `_phone` | `string` | Номер телефона |
| `_address` | `string` | Адрес доставки |

**Методы класса**

| Метод | Параметры | Возвращаемое значение | Описание |
|-------|-----------|----------------------|----------|
| `setPayment(payment)` | `payment: TPayment` | `void` | Устанавливает способ оплаты |
| `setEmail(email)` | `email: string` | `void` | Устанавливает email |
| `setPhone(phone)` | `phone: string` | `void` | Устанавливает номер телефона |
| `setAddress(address)` | `address: string` | `void` | Устанавливает адрес доставки |
| `setData(data)` | `data: Partial<IBuyer>` | `void` | Сохраняет одно или несколько полей покупателя (обновляет только переданные свойства) |
| `getData()` | нет | `IBuyer` | Возвращает все данные покупателя |
| `clear()` | нет | `void` | Сбрасывает все поля в значения по умолчанию |
| `validate()` | нет | `Partial<Record<keyof IBuyer, string>> \| null` | Проверяет валидность каждого поля. Поле считается валидным, если оно не пустое. Возвращает объект с ошибками или `null`, если всё валидно. |

```typescript
type TPayment = 'cash' | 'card';  // пользовательский тип (type alias), который определяет возможные способы оплаты в приложении

interface IBuyerModel {
  payment: TPayment;   // Способ оплаты: 'cash' | 'card'
  email: string;       // Адрес электронной почты
  phone: string;       // Номер телефона
  address: string;     // Адрес доставки
}
```

### Интерфейсы для обмена с сервером

#### `IOrder` – данные для отправки заказа


Используется при вызове метода `sendOrder` класса `AppApi`.


```typescript
interface IOrder {
  payment: TPayment;   // Способ оплаты: 'cash' | 'card'
  email: string;       // Электронная почта покупателя
  phone: string;       // Номер телефона
  address: string;     // Адрес доставки
  total: number;       // Итоговая сумма заказа (рассчитывается из корзины)
  items: string[];     // Массив идентификаторов товаров, которые покупает пользователь
}
```

### Интерфейс `IOrderResult`

**Назначение:** описывает структуру ответа сервера после успешного оформления заказа. Возвращается методом `sendOrder` класса `AppApi`.

```typescript
interface IOrderResult {
  id: string;    // уникальный идентификатор созданного заказа
  total: number; // итоговая сумма заказа (дублирует значение, переданное в IOrder)
}
```
