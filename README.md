# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.ф

## Модели данных

Модели данных отвечают за хранение и работу с данными в приложении. Ниже описаны три класса, реализующие эту функциональность.

### 1. Класс `ProductCatalog` (каталог товаров)

**Назначение и зона ответственности**  
Отвечает за хранение списка всех доступных товаров, а также за управление выбранным для детального просмотра товаром.

**Конструктор**  
Не принимает параметров. При создании экземпляра инициализирует пустой массив товаров и `null` для выбранного товара.

**Поля класса**

| Поле | Тип | Описание |
|------|-----|----------|
| `_products` | `IProduct[]` | Массив всех товаров, доступных в приложении |
| `_selectedProduct` | `IProduct \| null` | Товар, выбранный для подробного отображения |

**Методы класса**

| Метод | Параметры | Возвращаемое значение | Описание |
|-------|-----------|----------------------|----------|
| `setProducts(products)` | `products: IProduct[]` | `void` | Сохраняет переданный массив товаров в поле `_products` |
| `getProducts()` | нет | `IProduct[]` | Возвращает массив всех товаров |
| `getProductById(id)` | `id: string` | `IProduct \| undefined` | Возвращает товар по указанному идентификатору |
| `setSelectedProduct(product)` | `product: IProduct` | `void` | Сохраняет товар для подробного отображения |
| `getSelectedProduct()` | нет | `IProduct \| null` | Возвращает выбранный для просмотра товар |

**Интерфейс**

```typescript
interface IProduct {
  id: string;          // уникальный идентификатор товара
  description: string; // подробное описание
  image: string;       // URL изображения
  title: string;       // название товара
  category: string;    // категория (например, "электроника")
  price: number | null;// цена (null означает бесплатно или не указана)
}
```

### 2. Класс `Cart` (корзина)

**Назначение и зона ответственности**  
Управляет списком товаров, которые пользователь добавил для покупки. Обеспечивает добавление, удаление, подсчёт стоимости и количества товаров в корзине.

**Конструктор**  
Не принимает параметров. Инициализирует пустой массив товаров.

**Поля класса**

| Поле | Тип | Описание |
|------|-----|----------|
| `_items` | `IProduct[]` | Массив товаров, добавленных в корзину |

**Методы класса**

| Метод | Параметры | Возвращаемое значение | Описание |
|-------|-----------|----------------------|----------|
| `getItems()` | нет | `IProduct[]` | Возвращает массив товаров в корзине |
| `addItem(product)` | `product: IProduct` | `void` | Добавляет товар в корзину |
| `removeItem(productId)` | `productId: string` | `void` | Удаляет товар с указанным `id` из корзины |
| `clear()` | нет | `void` | Очищает корзину (удаляет все товары) |
| `getTotalPrice()` | нет | `number` | Возвращает сумму цен всех товаров в корзине (цены `null` трактуются как 0) |
| `getItemCount()` | нет | `number` | Возвращает количество товаров в корзине |
| `contains(productId)` | `productId: string` | `boolean` | Проверяет, присутствует ли товар с указанным `id` в корзине |

```typescript
interface ICart {
  getItems(): IProduct[];               // Получить все товары в корзине
  addItem(product: IProduct): void;     // Добавить товар
  removeItem(productId: string): void;  // Удалить товар по id
  clear(): void;                        // Очистить корзину
  getTotalPrice(): number;              // Общая стоимость товаров в корзине
  getItemCount(): number;               // Количество товаров
  contains(productId: string): boolean; // Проверить наличие товара по id
}
```

**Конструктор**

| `_address` | `string` | Адрес доставки |

**Методы класса**

| Метод | Параметры | Возвращаемое значение | Описание |

| `setPhone(phone)` | `phone: string` | `void` | Устанавливает номер телефона |

| `clear()` | нет | `void` | Сбрасывает все поля в значения по умолчанию |
| `validate()` | нет | `Partial<Record<keyof IBuyer, string>> \| null` | Проверяет валидность каждого поля. Поле считается валидным, если оно не пустое. Возвращает объект с ошибками или `null`, если всё валидно. |

interface IBuyerModel {

## Слой представления (View)

### Абстрактный класс `Component`

**Назначение:** базовый абстрактный класс для всех компонентов пользовательского интерфейса. Предоставляет набор утилитарных методов для работы с DOM и единый интерфейс рендеринга.

**Конструктор:** `protected constructor(protected readonly container: HTMLElement)`

**Параметры конструктора:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| `container` | `HTMLElement` | Корневой DOM-элемент компонента |

**Поля класса:**

| Поле | Тип | Описание |
|------|-----|----------|
| `container` | `HTMLElement` | Корневой DOM-элемент компонента (защищённое, только для чтения) |

**Защищённые методы (утилиты для работы с DOM):**

| Метод | Параметры | Описание |
|-------|-----------|----------|
| `setText` | `element: HTMLElement, value: unknown` | Устанавливает текстовое содержимое элемента. Если элемент существует, преобразует значение в строку |
| `setDisabled` | `element: HTMLElement, state: boolean` | Блокирует или разблокирует элемент (кнопку). При `state = true` добавляет атрибут `disabled` |
| `setHidden` | `element: HTMLElement` | Скрывает элемент, устанавливая `display: none` |
| `setVisible` | `element: HTMLElement` | Показывает элемент, удаляя `display: none` |
| `setImage` | `element: HTMLImageElement, src: string, alt?: string` | Устанавливает изображению src и альтернативный текст |

**Публичные методы:**

| Метод | Параметры | Возвращаемое значение | Описание |
|-------|-----------|----------------------|----------|
| `render` | `data?: Partial<T>` | `HTMLElement` | Обновляет компонент с переданными данными и возвращает корневой DOM-элемент. Данные применяются через `Object.assign` |

**Особенности реализации:**
- Класс является **абстрактным** и не может быть инстанциирован напрямую. Все компоненты представления должны наследоваться от него
- Используется **дженерик** `T`, который определяет тип данных, ожидаемых компонентом при рендеринге
- Метод `render` автоматически присваивает свойства объекта `data` текущему экземпляру класса, что позволяет использовать сеттеры для обновления DOM
- Все DOM-операции безопасно проверяют существование элементов перед манипуляцией

### Класс `Modal`

**Назначение:** управляет отображением модального окна. Обеспечивает открытие, закрытие и установку содержимого модального окна. Генерирует события при изменении состояния.

**Конструктор:** `constructor(container: HTMLElement, events: IEvents)`

**Параметры конструктора:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| `container` | `HTMLElement` | Корневой DOM-элемент модального окна (с классом `modal`) |
| `events` | `IEvents` | Брокер событий для генерации событий |

**Поля класса:**

| Поле | Тип | Описание |
|------|-----|----------|
| `closeButton` | `HTMLElement` | Кнопка закрытия модального окна (класс `modal__close`) |
| `contentElement` | `HTMLElement` | Контейнер для динамического содержимого (класс `modal__content`) |

**Свойства (геттеры/сеттеры):**

| Свойство | Тип | Описание |
|----------|-----|----------|
| `content` (сеттер) | `HTMLElement` | Устанавливает содержимое модального окна, заменяя все дочерние элементы |

**Методы:**

| Метод | Параметры | Возвращаемое значение | Описание |
|-------|-----------|----------------------|----------|
| `open()` | нет | `void` | Открывает модальное окно (добавляет класс `modal_active`) и генерирует событие `modal:open` |
| `close()` | нет | `void` | Закрывает модальное окно (удаляет класс `modal_active`), очищает содержимое и генерирует событие `modal:close` |

**Генерируемые события:**

| Событие | Когда генерируется | Данные |
|---------|-------------------|--------|
| `modal:open` | При открытии модального окна | – |
| `modal:close` | При закрытии модального окна | – |

**Особенности реализации:**
- Для поиска элементов используется `ensureElement`, гарантирующий наличие кнопки закрытия и контейнера контента
- Клик на оверлей (фон модального окна) также закрывает окно
- При закрытии содержимое модального окна полностью очищается через `innerHTML = ''`
- Модальное окно активируется/деактивируется CSS-классом `modal_active`

### Класс `Form`

**Назначение:** базовый класс для всех форм приложения. Обеспечивает обработку ввода данных, валидацию и управление состоянием кнопки отправки.

**Конструктор:** `constructor(container: HTMLFormElement, protected events: IEvents)`

**Параметры конструктора:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| `container` | `HTMLFormElement` | DOM-элемент формы |
| `events` | `IEvents` | Брокер событий для генерации событий |

**Поля:**

| Поле | Тип | Описание |
|------|-----|----------|
| `formElement` | `HTMLFormElement` | DOM-элемент формы |
| `submitButton` | `HTMLButtonElement` | Кнопка отправки |
| `errorsElement` | `HTMLElement` | Контейнер для ошибок |

**Сеттеры:**

| Сеттер | Тип | Описание |
|--------|-----|----------|
| `valid` | `boolean` | Блокирует/разблокирует кнопку отправки |
| `errors` | `string` | Устанавливает текст ошибок |

**События:**

| Событие | Данные | Описание |
|---------|--------|----------|
| `${name}:change` | `{ field: keyof T, value: string }` | Изменение поля формы |
| `${name}:form-submit` | — | Отправка формы ||


> **Примечание:** `${name}` — это значение атрибута `name` у формы. Например, для формы с `name="order"` события будут `order:change` и `order:form-submit`.

**Особенности реализации:**
- Для поиска обязательных элементов используется `ensureElement`, который гарантирует наличие кнопки отправки и контейнера для ошибок
- При отправке формы стандартное поведение (перезагрузка страницы) отменяется через `e.preventDefault()`
- Событие `input` отслеживает любые изменения полей формы и генерирует событие с именем поля и новым значением
- Класс является дженериком `Form<T>`, где `T` — тип данных формы (содержит названия полей и их типы)

### Класс `ContactsForm`

**Конструктор:** `constructor(container: HTMLFormElement, events: IEvents)`

**Поля:**

| Поле | Тип | Описание |
|------|-----|----------|
| `emailInput` | `HTMLInputElement` | Поле ввода email |
| `phoneInput` | `HTMLInputElement` | Поле ввода телефона |

**Сеттеры:**

| Сеттер | Тип | Описание |
